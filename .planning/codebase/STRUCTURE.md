## Directory Purposes

**`cinatra/`:**
- Purpose: The published payload of the extension. Only files under `cinatra/**` are included in the npm package (per `package.json` `"files"` field).
- Contains: `oas.json` — the complete agent flow definition in Cinatra AgentSpec format.
- Key files: `cinatra/oas.json`

**`.github/workflows/`:**
- Purpose: GitHub Actions CI/CD pipelines for the extracted extension repo.
- Contains: `ci.yml` (build + kind-gates), `release.yml` (publish).
- Key files: `.github/workflows/ci.yml`, `.github/workflows/release.yml`

**`.planning/codebase/`:**
- Purpose: GSD codebase map documents generated by `/gsd-map-codebase`.
- Generated: Yes (by GSD tooling).
- Committed: Yes.

## Key File Locations

**Entry Points:**
- `cinatra/oas.json`: The agent's declarative flow definition consumed by the Cinatra platform runtime. This is the primary artifact of the repo.
- `extension-kind-gate.mjs`: CLI entry point for CI validation (`node extension-kind-gate.mjs --package-root .`).

**Configuration:**
- `package.json`: Package name (`@cinatra-ai/auditor-agent`), version, `cinatra.kind = "agent"`, `cinatra.apiVersion`, `files` list.
- `tsconfig.json`: TypeScript compiler config (referenced by CI typecheck step; no `.ts` sources currently tracked).
- `.npmrc`: Registry configuration (existence only; contents not read).

**Core Logic:**
- `cinatra/oas.json`: All agent behavior is declared here — nodes (`resolve_skills`, `run_skills`, `review_gate`, `apply_patches`), control-flow edges, data-flow edges, and node metadata.
- `extension-kind-gate.mjs`: All CI gate logic — `parseArgs`, `validateAgent`, `validateWorkflow`, `validateWorkflowPackageShape`, `validateBpmnSanity`, `findWorkflowSidecars`, `runGate`.

**CI/CD:**
- `.github/workflows/ci.yml`: Runs on push/PR to `main`. Two jobs: `build` (classify → install → typecheck → test → pack dry-run) and `kind-gates` (OAS validation).
- `.github/workflows/release.yml`: Release/publish pipeline.

## Naming Conventions

**Files:**
- Cinatra manifest: `cinatra/oas.json` (fixed name required by platform convention).
- Gate script: `extension-kind-gate.mjs` (kebab-case `.mjs` ES module).
- Workflows: `ci.yml`, `release.yml` (kebab-case YAML).

**Directories:**
- `cinatra/`: Fixed name required by the Cinatra extension platform.
- `.github/workflows/`: Standard GitHub Actions location.

## Where to Add New Code

**Changing agent behavior (nodes, edges, inputs/outputs):**
- Edit `cinatra/oas.json` — add or modify nodes in `$referenced_components`, add edges in `control_flow_connections` and `data_flow_connections`, update `nodes` array and top-level `inputs`/`outputs`.

**Adding TypeScript source files (if the agent grows custom logic):**
- Place under a new `src/` directory (not yet present).
- Ensure `package.json` `files` field includes the compiled output directory.
- Add a `typecheck` script to `package.json` to enable the CI typecheck branch.

**Adding a new CI gate check:**
- Add a new exported pure function to `extension-kind-gate.mjs` returning `string[]`.
- Wire it into `runGate` or a new dispatch branch.

**Updating banned primitives list:**
- Edit `BANNED_PRIMITIVES` or `BANNED_TYPEHINTS` arrays in `extension-kind-gate.mjs` (lines 65–74).
